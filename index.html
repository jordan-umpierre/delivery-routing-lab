    How the algorithms work

    Dijkstra expands nodes in order of exact distance from the start, so the first time it settles the destination the path is provably shortest. A* adds a heuristic: the great-circle (haversine) distance to the destination. Because no street segment can be shorter than the straight line between its endpoints, the heuristic never overestimates the remaining distance — it is admissible — so A* returns the same optimal cost while typically expanding fewer nodes. For multi-stop requests, nearest-neighbor greedily orders the stops by shortest-path cost and 2-opt reverses segments of that order while doing so shortens the total route; the result is never worse than nearest-neighbor's, though neither guarantees the true optimum.
